# Vektorix implementation plan

## Repository audit

The repository contained no application source, package manifest, tests, or reusable UI. The installed project skills and Git history are retained. The product therefore starts from a clean Next.js architecture rather than refactoring absent code.

## Delivery phases

1. Establish strict Next.js, Tailwind, lint, test, Drizzle schema, recovery, error boundaries, and the project-specific skill.
2. Implement deterministic DNA, gesture extraction, quality selection, mutation, and fusion with unit tests.
3. Build the responsive R3F organism from a shader core, membrane, GPU points, camera rig, pointer plane, keyboard input, audio analyser, and bounded post-processing.
4. Connect the seven-stage anonymous lab flow to local recovery, naming, preview capture, publishing, and share URLs.
5. Build public Vektor, explore, and remix experiences with low-cost previews and ancestry.
6. Verify reduced motion, WebGL fallback, keyboard completion, mobile layout, build, lint, typecheck, unit tests, and browser scenarios.

## External-service boundary

The database and authentication schemas are production-oriented, but a deployed PostgreSQL instance, email provider, and object store require environment credentials. The local product uses an anonymous repository and encoded deterministic share payloads so creation and sharing remain testable without those services.
