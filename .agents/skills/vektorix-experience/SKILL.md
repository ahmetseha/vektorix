# Vektorix Experience

## Protect the product premise

- Build living “Vektor” organisms from procedural math, seeded randomness, gestures, local Web Audio analysis, and deterministic crossover.
- Never add AI, LLMs, prompt generation, generative APIs, blockchain, NFTs, tokens, wallets, or “powered by AI” language.
- Make meaningful interaction available within five seconds. Avoid onboarding carousels and generic configurator sliders.
- Keep the visual language alien, precise, dark, energetic, and editorial. Avoid generic SaaS dashboards, gradient blobs, glass-card grids, and ornamental icon density.

## Protect determinism and data

- Validate every DNA payload with the versioned Zod schema.
- Never use `Math.random()` for persistent organism traits. Derive values from the seed and normalized inputs.
- Store compact gesture summaries, never raw unbounded pointer streams or audio.
- Make mutation and fusion reproducible from parent DNA and a fusion seed. Preserve ancestry and create a new child record.
- Store DNA as the source of truth; treat images and videos as previews.

## Protect the render loop

- Keep per-frame values in refs, typed arrays, buffer attributes, or stable uniforms. Never write them to React or Zustand state each frame.
- Use one buffer-based particle system, bounded trail storage, reused resources, adaptive DPR, and quality tiers.
- Gate membrane, post-processing, trail density, and particle count by capability. Pause or reduce work when hidden.
- Keep bloom subtle, transparency intentional, camera motion bounded, and mobile framing wider.
- Verify APIs against installed versions and run performance checks after adding effects.

## Protect interaction and access

- Support mouse, pen, touch, and keyboard. Provide single-action alternatives to gesture-only behavior.
- Give the canvas a meaningful accessible name and expose stage/state changes through DOM text or live regions.
- Respect `prefers-reduced-motion`, provide a pause control, avoid flashing, and retain a static/WebGL fallback.
- Keep touch targets at least 44px where practical, focus visible, contrast high, and microphone use optional with a clear local-only explanation.

## Definition of done

- Run typecheck, lint, unit tests, build, and relevant Playwright flows.
- Verify desktop, mobile, reduced-motion, WebGL fallback, console errors, deterministic serialization, and share reconstruction.
- Report external-service limitations honestly; never present mocked or placeholder infrastructure as production-complete.
